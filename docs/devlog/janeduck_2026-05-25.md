#### 에러 처리 수정

TTS 생성 → Blob 업로드 → DB INSERT 전체를 try/catch로 감쌈. 수정 전에는 각 단계 실패 시 unhandled error로 터짐.

- 실패 시 503 반환, `console.error`로 원인 로깅
- 빈 audio_url이 DB에 박히는 케이스 원천 차단 (INSERT는 TTS + Blob 모두 성공 시에만 도달)

#### 로컬 테스트 — 환경변수 오타로 503

증상: 스피커 버튼 클릭 시 `ERR_INVALID_ARG_TYPE: Received undefined at Buffer.from`

원인: `.env.local`의 `GOOGLE_TTS_CREDENTIALS` 변수명 오타 → env var이 `undefined`로 읽힘 → `Buffer.from(undefined, "base64")` 에러

진단 방법: 임시 `console.log("[TTS] CREDENTIALS present:", !!process.env.GOOGLE_TTS_CREDENTIALS)` 추가 → `false` 확인 → 오타 수정 → 재시도 성공

진단 로그는 수정 확인 후 제거.
