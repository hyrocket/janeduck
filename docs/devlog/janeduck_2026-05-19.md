### 진도 미표시 버그 조사 및 수정 — user_id 불일치

**증상**: 로그인 후 Writing Mode 학습해도 `/decks` 화면에 진도 바가 표시 안 됨.

**조사 결과**:
- `/decks` 집계 SQL: SM-2 의존 없음, `mastery_level = 5` 기준만 사용 → 쿼리 자체는 문제없음
- Python 쪽: `uuid_mod.UUID(google_sub)` → `ValueError` → `load_context_node`·`update_srs_mastery_node`가 에러 조용히 삼킴 → **DB에 아무것도 저장 안 됨**
- Next.js 쪽: `session.user.id = token.sub` (Google OAuth sub, 숫자 문자열) → `WHERE user_id = "11720..."` (UUID 컬럼) → 쿼리 실패 or 0 반환

**수정**: `auth.ts` session/jwt 콜백에서 Google sub → **UUID v5 결정론적 변환**
```typescript
// UUID_NAMESPACE_URL로 uuid5("google:{sub}") 생성
token.userId = googleSubToUUID(token.sub)
session.user.id = token.userId
```
- 같은 Google 계정 → 항상 동일 UUID → Python `uuid_mod.UUID()` 성공, Next.js WHERE 절도 유효 UUID
- 구 JWT에도 `!token.userId` 조건으로 자동 마이그레이션 (재로그인 없이도 적용)

**추가**: `app/decks/page.tsx`에 `export const dynamic = "force-dynamic"` 추가 (캐시 방지)
