### 중앙 덱 라이브러리 + 사용자 덱 구조 전환

#### 개념

| 구분 | deck_type | 설명 |
|---|---|---|
| 라이브러리 덱 | `library` | JaneDuck 원본. `owner_id = SYSTEM_OWNER_ID`. 수정 불가. |
| My Deck | `user` | 사용자 사본. `owner_id = userId`. 수정 가능 구조. |

`SYSTEM_OWNER_ID = '00000000-0000-0000-0000-000000000001'` — `lib/system.ts`에 상수 정의.

#### DB 변경 (Migration 005)

- `decks`에 `deck_type TEXT CHECK ('library','user')`, `is_default BOOLEAN` 컬럼 추가
- 기존 null-owner 덱 2개 → `deck_type='library'`, `owner_id=SYSTEM_OWNER_ID`, `is_default=true`, "(test)" 제거
- 기존 user_cards가 있던 각 UUID마다 user 덱 사본 + 카드 전체 복사 + `user_cards.card_id` 업데이트
  - 총 13개 UUID, user_cards 98행 전원 보존
  - `writing_attempts` 삭제 없음

#### 설계 결정 사항

- `user_decks` 별도 테이블 안 만듦. `decks.owner_id`로 충분.
- `deck_type`이 단일 진실 출처 — owner가 누구든 `library`면 라이브러리에 보임 (향후 일반 유저도 라이브러리 기여 가능한 구조)
- 카드 복사 방식 (참조 아님) — 사용자가 자기 덱 카드를 독립적으로 수정·추가·삭제 가능한 구조
