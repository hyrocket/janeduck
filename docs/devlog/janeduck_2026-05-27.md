# JaneDuck 업무일지 — 2026-05-27

## 주요 진행

### TTS 연속 탭 버그 수정

#### 증상

단어 카드의 스피커 버튼을 연속으로 탭하면 사운드가 겹쳐서 들림.

#### 원인 분석

**버그 1 — 모바일 이중 호출**

`FlashCard.tsx`의 스피커 버튼에 `onClick`과 `onTouchEnd`가 동시에 등록되어 있었음.
모바일에서 탭 한 번에 `onTouchEnd` → `onClick` 순으로 둘 다 발화 → `handlePlayAudio` 2회 호출 → 오디오 두 번 재생.

**버그 2 — 재생 중 중복 재생 차단 없음**

URL이 이미 캐시된 경우, 재생 중에 다시 탭해도 막는 가드가 없었음.
`audioLoading` 가드는 fetch 중일 때만 작동하고, URL이 있을 때는 `new Audio(url).play()`를 바로 실행함.

**버그 3 — Writing Mode Action 이중 호출**

`handleAction`에 `phase` 체크가 없어, React 상태 업데이트 전 빠른 이중 탭 시 두 번 API 호출 가능.

#### 수정 내용

**FlashCard.tsx**

- `onTouchEnd={handlePlayAudio}` 제거 — `onClick`이 터치도 처리함
- `playingRef` 추가 — 재생 중 새 탭 차단
- `currentAudioRef` 추가 — 재생 전 이전 오디오 정지 후 교체

**WritingClient.tsx**

- `handleAction` 진입 조건에 `phase !== "awaiting_action"` 추가
